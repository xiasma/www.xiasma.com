Accessibility.

Xiasma treats accessibility as a baseline, not an upgrade. This statement explains where the site stands and how to tell us if something does not work for you.

Last updated: 2026.

Standard

www.xiasma.com is built to target the Web Content Accessibility Guidelines (WCAG) 2.2 at level AA.

How we approach it

  • Semantic HTML for headings, landmarks and navigation.
  • Visible focus styles on links, buttons and form fields.
  • Skip-to-content link as the first focusable element.
  • Keyboard-operable navigation, including the mobile menu.
  • Colour contrast that meets AA for body and interactive text.
  • Resizable text via standard browser controls (no fixed pixel viewports).
  • Reduced-motion support: animations and smooth scrolling are minimised when the operating system requests it.
  • Alt text and ARIA labels on meaningful images and icons; decorative graphics are hidden from assistive tech.

Known limitations

This is a small brochure site. We will list any specific known issues here as they are reported.

Tell us

If you find something inaccessible, or you would like content in an alternative format, please email info@xiasma.com. We aim to respond within a reasonable working timeframe and to fix issues quickly.
